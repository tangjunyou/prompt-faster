# Epic 指标记录（Metrics Tracker）

> 目的：在每个 Epic 结束时记录关键指标，避免“未追踪”。
> 规则：不写工期估算；没有数据则标记“未追踪”，但必须保留字段。

## 使用说明

- 每次 Epic 复盘后更新一行。
- 指标字段保持固定口径，便于横向比较。
- 若指标来源为日志/测试/CI，请在“来源”列注明文档或报告路径。

## 指标字段定义

- **计划点数 / 实际点数**：以故事点或团队约定的等价单位记录。
- **周期数**：以迭代/冲刺/阶段为单位。
- **阻塞数**：影响进度或交付的显著阻塞数量。
- **技术债条目**：复盘中记录的技术债数量。
- **测试/质量**：覆盖率、回归通过情况或质量摘要。
- **事故数**：生产或准生产事故。
- **目标达成**：`已达成/总目标`。
- **成功标准**：是否满足 Epic 设定的完成标准（如“达成/部分/未达成”）。
- **反馈摘要**：利益相关方或自验反馈。
